# 분석 데이터 추가 마이그레이션

다른 PC의 SQLite `greed.db`에 저장된 분석 결과만 현재 MariaDB에 추가하는 절차입니다.

## 대상 데이터

- 가져오는 데이터: `analyses`
- 가져오지 않는 데이터: `runs`, `analysis_jobs`, `stock_prices`, `krx_stocks`, `us_stocks`
- 가져온 모든 분석은 MariaDB의 현재 최대 `runs.id`에 연결됩니다.
- 중복 검사는 하지 않습니다. 같은 분석이 이미 있어도 그대로 추가됩니다.

## 사전 준비

1. 현재 MariaDB를 백업합니다.
2. 다른 PC의 `greed.db`를 이 PC로 복사합니다.
   - 예: `C:\work\greed\other_greed.db`
3. MariaDB 접속 정보가 `.env`의 `DATABASE_URL`에 설정되어 있는지 확인합니다.
4. `PyMySQL`이 설치되어 있는지 확인합니다.

```powershell
.venv\Scripts\python.exe -m pip show PyMySQL
```

설치되어 있지 않으면 백엔드 의존성을 설치합니다.

```powershell
.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```

## Dry-run

먼저 rollback 모드로 가져올 건수와 대상 run을 확인합니다.

```powershell
.venv\Scripts\python.exe scripts\import_sqlite_analyses_to_mariadb.py `
  --source C:\work\greed\other_greed.db `
  --dry-run
```

출력 예시:

```text
[DRY-RUN] analysis import completed
  - target_run_id: 4
  - source_analyses: 92
  - inserted_analyses: 92
```

## 실제 실행

dry-run 결과가 맞으면 `--dry-run`을 빼고 실행합니다.

```powershell
.venv\Scripts\python.exe scripts\import_sqlite_analyses_to_mariadb.py `
  --source C:\work\greed\other_greed.db
```

특정 MariaDB URL을 직접 지정하려면 `--target`을 사용합니다.

```powershell
.venv\Scripts\python.exe scripts\import_sqlite_analyses_to_mariadb.py `
  --source C:\work\greed\other_greed.db `
  --target "mysql+pymysql://USER:PASSWORD@HOST:3306/greed?charset=utf8mb4"
```

## 확인

1. 웹 앱에서 가장 큰 run ID의 분석 목록을 엽니다.
2. 추가된 종목들이 보이는지 확인합니다.
3. 필요하면 MariaDB에서 `analyses` 건수를 확인합니다.

```sql
SELECT MAX(id) FROM runs;
SELECT COUNT(*) FROM analyses;
```

## 실패 시 확인할 점

- `Target DB has no runs`: MariaDB에 run이 하나도 없습니다. 웹 또는 API로 run을 먼저 생성하세요.
- `Source analyses table is missing required columns`: 다른 PC의 SQLite 스키마가 현재 앱 스키마와 다릅니다. 같은 버전의 앱에서 생성된 DB인지 확인하세요.
- `No module named 'pymysql'`: `backend\requirements.txt` 기준으로 의존성을 설치하세요.
