# 모바일 기능 정의

> 기준: 2026-05-09 기획

## 1. 개요

GREED 모바일은 태블릿과 스마트폰에서 분석 결과와 종목 요약을 조회하는 개인 전용 앱이다.  
쓰기 기능은 포함하지 않으며, 조회 전용 백엔드(`backend-mobile`)를 별도로 운영한다.

### 구성 요소

| 구성 요소 | 위치 | 역할 |
|---|---|---|
| `backend-mobile/` | 신규 | 조회 전용 FastAPI 서버. OCI MariaDB 연결 |
| `mobile/` | 신규 | Expo React Native 앱 (iOS / Android) |
| `backend/` | 기존 | 변경 없음 |
| `frontend/` | 기존 | 변경 없음 |

### 아키텍처

```
mobile app
  └─ HTTPS + X-API-Key 헤더
       └─ backend-mobile (FastAPI, 조회 전용)
            └─ OCI MariaDB (읽기 전용 쿼리)
```

기존 `backend`와 `backend-mobile`은 동일한 OCI MariaDB를 바라본다.  
`backend-mobile`은 `backend`와 완전히 독립적으로 배포되며 서로 영향을 주지 않는다.

## 2. backend-mobile

### 2-1. 구조

```
backend-mobile/
├── main.py           # FastAPI 앱 진입점, X-API-Key 전역 미들웨어
├── database.py       # OCI MariaDB SQLAlchemy 세션
├── models.py         # runs / analyses / krx_stocks / us_stocks 모델 (기존과 동일)
├── schemas.py        # 모바일 응답 전용 Pydantic 스키마
├── auth.py           # API Key 검증 Depends
├── crud.py           # 조회 전용 쿼리
└── routers/
    ├── analyses.py   # /analyses, /analyses/{id}
    └── stocks.py     # /stocks/summary
```

### 2-2. 엔드포인트

모든 엔드포인트는 GET 메서드만 제공한다.

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/analyses` | 분석 목록 (필터, 페이지네이션) |
| GET | `/analyses/{id}` | 분석 상세 (마크다운 포함) |
| GET | `/stocks/summary` | 종목별 매수/홀드/매도 집계 |

#### GET /analyses 파라미터

| 파라미터 | 타입 | 설명 |
|---|---|---|
| `judgment` | string | 매수 / 홀드 / 매도 필터 |
| `q` | string | 종목명, 초성, 티커 검색 |
| `page` | int | 페이지 번호 (기본 1) |
| `per_page` | int | 페이지 크기 (기본 20, 최대 50) |

### 2-3. 보안

- 모든 요청에 `X-API-Key` 헤더가 있어야 한다.
- 서버는 `.env`에 저장된 `MOBILE_API_KEY` 값과 헤더 값을 비교해 일치하지 않으면 401을 반환한다.
- 서버는 HTTPS를 통해서만 접근을 허용한다. OCI 로드밸런서 또는 nginx에서 TLS를 종료한다.
- POST / PUT / DELETE 엔드포인트는 구현하지 않는다.

### 2-4. 데이터베이스 연결

- `.env`의 `DATABASE_URL`을 그대로 사용한다.
- 연결 실패 시 서버 기동을 중단한다. SQLite 폴백은 지원하지 않는다.
- 쓰기 트랜잭션을 발생시키지 않으므로 DB 사용자 계정은 SELECT 권한만 부여해도 된다.

## 3. mobile 앱

### 3-1. 기술 스택

| 항목 | 선택 |
|---|---|
| 프레임워크 | Expo (React Native) |
| 언어 | TypeScript |
| 라우팅 | expo-router (파일 기반) |
| 서버 상태 | @tanstack/react-query |
| HTTP | axios |
| 마크다운 | react-native-markdown-display |
| API 키 저장 | expo-secure-store |
| 태블릿 감지 | react-native `useWindowDimensions` |

### 3-2. 폴더 구조

```
mobile/
├── app/
│   ├── _layout.tsx              # 루트 레이아웃, API 키 초기화
│   ├── (tabs)/
│   │   ├── _layout.tsx          # 하단 탭 바
│   │   ├── index.tsx            # Analyses 화면
│   │   └── stocks.tsx           # Stocks 화면
│   └── analysis/
│       └── [id].tsx             # 분석 상세 화면
├── components/
│   ├── AnalysisCard.tsx         # 목록 카드
│   ├── JudgmentBadge.tsx        # 매수/홀드/매도 뱃지
│   ├── FilterBar.tsx            # 판정 필터 칩
│   └── MarkdownView.tsx         # 마크다운 렌더러
├── api/
│   ├── client.ts                # axios 인스턴스, X-API-Key 인터셉터
│   ├── analyses.ts              # 분석 조회 훅
│   └── stocks.ts                # 종목 요약 조회 훅
└── hooks/
    └── useTablet.ts             # 태블릿 여부 (width ≥ 768)
```

### 3-3. 화면 정의

#### Analyses 화면

- 시스템은 분석 목록을 최신순으로 보여준다.
- 사용자는 매수 / 홀드 / 매도 필터 칩을 선택해 판정 기준으로 목록을 좁힐 수 있다.
- 사용자는 검색창에 종목명, 초성, 티커를 입력해 목록을 검색할 수 있다.
- 목록을 끝까지 스크롤하면 다음 페이지를 자동으로 불러온다.
- 사용자는 아래로 당겨 목록을 새로고침할 수 있다.
- 목록 항목에는 종목명, 티커, 매매 판정, 추세, 구름대 위치, 이동평균 배열, 생성 시각을 표시한다.
- 사용자는 항목을 탭해 분석 상세 화면으로 이동한다.
- 태블릿에서는 화면을 좌우로 나누어 왼쪽에 목록, 오른쪽에 상세를 동시에 표시한다.
- 스마트폰에서는 항목 탭 시 상세 화면으로 전환한다.

#### Stocks 화면

- 시스템은 분석이 저장된 종목을 종목 단위로 모아 보여준다.
- 각 종목 카드에는 종목명, 티커, 매수/홀드/매도 분석 개수, 가장 최근 분석 시각을 표시한다.
- 사용자는 검색창으로 종목명, 초성, 티커를 검색할 수 있다.
- 사용자는 종목을 탭하면 해당 종목의 분석만 필터링한 Analyses 화면으로 이동한다.

#### 분석 상세 화면

- 시스템은 분석 마크다운 전문을 화면에 렌더링해 보여준다.
- 화면 상단에는 종목명, 티커, 매매 판정, 생성 시각을 표시한다.
- 시스템은 추세, 구름대 위치, 이동평균 배열을 뱃지로 요약해 상단에 표시한다.
- 사용자는 뒤로가기로 이전 화면으로 돌아간다.
- 태블릿 split 레이아웃에서는 별도 화면 전환 없이 우측 패널을 갱신한다.

### 3-4. 보안

- 앱 최초 실행 시 API 키 입력 화면을 표시한다.
- 입력된 API 키는 `expo-secure-store`에 암호화 저장되며 이후 입력을 생략한다.
- 모든 API 요청 헤더에 `X-API-Key: <저장된 키>`를 자동으로 포함한다.
- 401 응답을 받으면 API 키 입력 화면으로 이동한다.
- 앱 설정에서 API 키를 초기화할 수 있다.

### 3-5. 레이아웃 분기 기준

| 조건 | 레이아웃 |
|---|---|
| 화면 너비 ≥ 768dp | 태블릿 (split) |
| 화면 너비 < 768dp | 스마트폰 (stack) |

---

## 4. OCI 배포

- `backend-mobile`은 기존 `backend`와 다른 포트 또는 다른 서브도메인으로 배포한다.
- OCI 로드밸런서 또는 nginx에서 HTTPS를 처리하고 HTTP 요청은 HTTPS로 리다이렉트한다.
- `backend-mobile` 프로세스는 `systemd` 또는 `supervisor`로 관리한다.
- DB 연결 계정에는 SELECT 권한만 부여한다.

---

## 5. 구현 범위 제외

- 분석 생성, 수정, 삭제는 모바일 앱에서 제공하지 않는다.
- 분석 잡 생성 및 조회는 제공하지 않는다.
- 종목 검색(자동완성)은 제공하지 않는다.
- 가격 새로고침은 제공하지 않는다.
- 실행(runs) 목록 조회는 제공하지 않는다.
