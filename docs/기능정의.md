# 기능 정의

> 기준: 2026-05-07 현재 구현된 동작

## 1. 종목 선별과 주봉 데이터 생성

### 1-1. 전종목 스크리닝

- 사용자는 KOSPI, KOSDAQ 종목을 대상으로 기술적 조건에 맞는 종목을 선별할 수 있다.
- 시스템은 일목 구름 돌파, 돌파 시 거래량 증가, 이동평균선 구름 돌파, 골든크로스 여부를 기준으로 후보 종목을 찾는다.
- 시스템은 우선주, 스팩, 리츠, 일정 시가총액 미만 종목을 제외해 분석 대상의 품질을 높인다.
- 스크리닝은 중간 진행 상태를 보존하므로 같은 날 다시 실행해도 이미 처리한 종목을 이어서 진행할 수 있다.
- 스크리닝이 완료되면 조건을 만족한 종목 목록과 각 종목의 분석용 주봉 CSV를 결과로 남긴다.

### 1-2. 한국/미국 단일 종목 주봉 생성

- 사용자는 한국 종목 코드 또는 미국 티커를 기준으로 단일 종목 주봉 데이터를 생성할 수 있다.
- 생성되는 데이터에는 시가, 고가, 저가, 종가, 거래량, 거래대금, 이동평균, ATR, RSI, MACD, 다이버전스, 일목균형표 지표가 포함된다.
- 시스템은 향후 구름 방향을 판단할 수 있도록 미래 구름 전용 26주 구간을 함께 포함한다.
- 미국 종목은 NASDAQ, NYSE, AMEX listing에서 종목명을 조회하고 금요일 기준 주봉으로 집계한다.
- 이 데이터는 이후 AI 에이전트 또는 사용자가 주봉 기술적 분석을 작성하는 입력 자료로 사용된다.

## 2. 종목 검색과 티커 관리

- 사용자는 웹에서 종목명, 초성, 한국 종목 코드, 미국 티커로 종목을 검색할 수 있다.
- 시스템은 KRX 종목과 US 종목을 함께 검색하고, 각 결과에 코드, 종목명, 시장 구분을 제공한다.
- 숫자 한국 종목 코드는 6자리로 정규화한다.
- 사용자는 종목 검색 결과를 선택해 비동기 분석을 시작할 수 있다.
- 운영자는 `refresh_tickers.py`로 KRX/US 종목 목록을 DB에 갱신할 수 있다.

## 3. 분석 실행 단위 관리

### 3-1. 실행 생성

- 사용자는 분석 작업을 묶는 실행 단위를 새로 만들 수 있다.
- 실행에는 선택적으로 메모를 남길 수 있다.
- 시스템은 각 실행에 고유 ID와 생성 시각을 부여한다.
- 사용자는 실행 목록에서 각 실행의 생성 시각, 메모, 저장된 분석 개수를 확인할 수 있다.

### 3-2. 실행별 분석 관리

- 사용자는 특정 실행에 여러 종목의 분석 결과를 저장할 수 있다.
- 저장된 분석은 해당 실행에 연결되어 나중에 실행 단위로 다시 조회할 수 있다.
- 같은 종목이라도 서로 다른 실행에 여러 번 저장할 수 있다.

## 4. 분석 결과 저장

### 4-1. CLI 기반 저장

- 사용자는 외부 분석 흐름에서 생성된 마크다운 분석 결과를 CLI로 저장할 수 있다.
- 여러 분석 파일을 한 번에 저장할 수 있다.
- 시스템은 분석 결과에서 핵심 판단 정보를 추출해 검색과 요약에 사용할 수 있도록 함께 저장한다.
- 저장에 성공하면 종목별 성공 여부와 최종 매매 판정을 확인할 수 있다.
- 저장에 실패한 분석은 어떤 항목이 부족했는지 확인할 수 있다.

### 4-2. 웹 트리거 기반 비동기 분석

- 사용자는 웹에서 종목과 실행 ID, 모델을 선택해 분석 잡을 생성할 수 있다.
- 지원 모델 선택값은 Claude, Codex, Gemini이다.
- 시스템은 잡을 즉시 `pending` 상태로 저장하고 백그라운드에서 주봉 CSV 생성, 모델 CLI 분석, 마크다운 검증, DB 저장을 순서대로 처리한다.
- 동일한 날짜의 종목 CSV가 있으면 시스템은 차트 캐시를 재사용해 반복 분석 속도를 높인다.
- 사용자는 잡 상태를 폴링해 `done` 또는 `failed` 여부를 확인할 수 있다.
- 성공한 잡은 생성된 분석 ID를 제공하므로 사용자가 분석 상세 화면으로 이동할 수 있다.
- 실패한 잡은 `pick`, `model_start`, `model_exit`, `parser`, `timeout`, `db` 중 어느 단계에서 실패했는지와 사유를 저장한다.

### 4-3. 웹 수동 입력

- 사용자는 웹 화면에서 분석 결과를 직접 등록할 수 있다.
- 등록 과정은 모델 선택, 저장할 실행 선택 또는 새 실행 생성, 분석 내용 입력 순서로 진행된다.
- 사용자는 종목 코드, 종목명, 모델, 분석 마크다운을 입력한다.
- 입력 중 시스템은 분석 내용에서 핵심 항목을 읽을 수 있는지 미리 보여준다.
- 저장 시점에도 시스템이 다시 검증하며, 필수 항목이 부족하면 저장하지 않는다.

## 5. 분석 내용 검증

### 5-1. 필수 항목

분석 결과는 저장 전에 다음 항목을 포함해야 한다.

- 매매 판정: 매수, 홀드, 매도 중 하나
- 추세: 상승, 하락, 횡보 중 하나
- 구름대 위치: 구름 위, 구름 안, 구름 아래 중 하나
- 이동평균 배열: 정배열, 역배열, 혼조 중 하나

### 5-2. 가격 항목

- 시스템은 분석 내용에서 진입가, 1차 목표가, 손절가를 읽어 요약 정보로 저장한다.
- 가격 항목은 없을 수 있으며, 없더라도 필수 항목이 모두 있으면 저장할 수 있다.
- 가격이 범위로 적혀 있으면 시스템은 최소값과 최대값을 함께 저장한다.
- 눌림 진입과 돌파 진입이 별도 행으로 적혀 있으면 시스템은 각각을 진입 후보로 인식한다.
- 매수 또는 홀드 판정에서 목표가가 진입가보다 낮거나 손절가가 진입가보다 높으면 저장을 거부한다.

### 5-3. 검증 실패 처리

- 필수 항목이 누락되면 시스템은 분석 저장을 거부한다.
- 가격 관계가 맞지 않으면 시스템은 `price_consistency` 실패로 저장을 거부한다.
- 사용자는 누락 또는 실패 항목을 확인한 뒤 분석 내용을 수정해 다시 저장할 수 있다.
- 선택 가격 항목만 없거나 읽을 수 없는 경우에는 해당 가격을 비워 둔 채 저장한다.

## 6. 실행 목록 조회

- 사용자는 전체 실행 목록을 최신순으로 볼 수 있다.
- 각 실행에서는 생성 시각, 메모, 저장된 분석 개수를 확인할 수 있다.
- 사용자는 목록에서 실행을 선택해 해당 실행의 종목 분석 목록으로 이동할 수 있다.
- 실행이 없을 때는 새 실행을 만들 수 있다.

## 7. 실행별 종목 분석 조회와 분석 시작

- 사용자는 특정 실행에 포함된 종목 분석 목록을 볼 수 있다.
- 목록에서는 종목명, 종목 코드, 매매 판정, 추세, 구름대 위치, 이동평균 배열, 모델을 확인할 수 있다.
- 사용자는 매매 판정 기준으로 목록을 필터링할 수 있다.
- 종목을 선택하면 해당 분석의 상세 화면으로 이동한다.
- 사용자는 이 화면에서 종목 검색 후 모델을 선택해 새 분석 잡을 시작할 수 있다.
- 사용자는 이 화면에서 새 분석을 수동으로 추가할 수 있다.

## 8. 전체 분석 조회

- 사용자는 모든 실행에 저장된 분석을 최신순으로 볼 수 있다.
- 목록에서는 종목명, 종목 코드, 소속 실행, 매매 판정, 생성 시각, 사용 모델을 확인할 수 있다.
- 사용자는 매매 판정, 실행 ID, 종목명, 초성, 티커 기준으로 분석 목록을 필터링할 수 있다.
- 사용자는 현재가와 진입가의 괴리율이 일정 기준 이내인 분석만 볼 수 있다.
- 사용자는 전체 진입 후보, 눌림 진입, 돌파 진입 기준으로 진입가 근접 분석을 필터링할 수 있다.
- 필터를 초기화해 전체 목록으로 돌아갈 수 있다.

## 9. 분석 상세 조회

- 사용자는 저장된 분석 마크다운 전문을 읽을 수 있다.
- 시스템은 분석 상단에 종목 코드, 종목명, 소속 실행, 사용 모델, 생성 시각, 매매 판정을 보여준다.
- 시스템은 추세, 구름대 위치, 이동평균 배열을 핵심 지표로 요약해 보여준다.
- 사용자는 종목 코드를 클립보드에 복사할 수 있다.
- 같은 종목의 과거 분석 이력이 있으면 최신순으로 보여주고, 사용자는 다른 시점의 분석으로 이동할 수 있다.
- 사용자는 상세 화면에서 같은 종목을 현재 실행에 다시 분석하도록 빠른 분석 잡을 시작할 수 있다.

## 10. 가격 레벨 확인

- 분석 상세 화면에서 사용자는 해당 종목의 최신 종가를 확인할 수 있다.
- 시스템은 최신 종가와 분석에 포함된 진입가, 목표가, 손절가를 함께 보여준다.
- 시스템은 현재가 대비 각 가격 레벨의 차이를 퍼센트로 표시한다.
- 가격이 범위로 저장된 경우 하단과 상단을 함께 표시한다.
- 사용자는 가격 정보를 수동으로 새로고침할 수 있다.
- 최신 가격을 가져올 수 없는 경우에도 분석 상세 내용은 계속 확인할 수 있다.

## 11. 잡 목록 조회

- 사용자는 전체 분석 잡 목록을 최신순으로 볼 수 있다.
- 시스템은 각 잡의 ID, 티커, 실행 ID, 모델, 상태, 생성 시각을 보여준다.
- 사용자는 pending 또는 failed 상태 기준으로 잡 목록을 필터링할 수 있다.
- 완료된 잡은 연결된 분석 상세 화면으로 이동할 수 있다.
- 실패한 잡은 실패 메시지와 원본 마크다운이 있으면 함께 확인할 수 있다.

## 12. 종목별 요약 조회

- 사용자는 분석이 저장된 종목을 종목 단위로 모아 볼 수 있다.
- 시스템은 종목별 매수, 홀드, 매도 분석 개수와 최신 분석 시각을 보여준다.
- 사용자는 종목명, 초성, 티커로 종목 요약 목록을 검색할 수 있다.
- 사용자는 판정 개수나 최신 분석 시각 기준으로 목록을 정렬할 수 있다.
- 사용자는 특정 종목을 선택해 해당 종목의 분석 목록으로 이동할 수 있다.

## 13. 현재 구현 범위

- 실행 목록, 실행별 분석 목록, 전체 분석 목록, 분석 상세 조회는 사용할 수 있다.
- 분석 수동 입력, CLI 저장, 웹 비동기 분석 잡 생성은 사용할 수 있다.
- Claude, Codex, Gemini 기반 웹 분석 트리거를 사용할 수 있다.
- 한국/미국 종목 검색과 티커 목록 갱신을 사용할 수 있다.
- 최신 종가 기반 가격 레벨 확인과 가격 새로고침은 분석 상세에서 사용할 수 있다.
- 잡 목록과 종목별 요약 화면을 사용할 수 있다.
